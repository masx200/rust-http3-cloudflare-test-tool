29/// ```text
30/// 5. ZONE FILES
31///
32/// Zone files are text files that contain RRs in text form.  Since the
33/// contents of a zone can be expressed in the form of a list of RRs a
34/// Zone File is most often used to define a zone, though it can be used
35/// to list a cache's contents.  Hence, this section first discusses the
36/// format of RRs in a Zone File, and then the special considerations when
37/// a Zone File is used to create a zone in some name server.
38///
39/// 5.1. Format
40///
41/// The format of these files is a sequence of entries.  Entries are
42/// predominantly line-oriented, though parentheses can be used to continue
43/// a list of items across a line boundary, and text literals can contain
44/// CRLF within the text.  Any combination of tabs and spaces act as a
45/// delimiter between the separate items that make up an entry.  The end of
46/// any line in the Zone File can end with a comment.  The comment starts
47/// with a ";" (semicolon).
48///
49/// The following entries are defined:
50///
51///     <blank>[<comment>]
52///
53///     $ORIGIN <domain-name> [<comment>]
54///
55///     $INCLUDE <file-name> [<domain-name>] [<comment>]
56///
57///     <domain-name><rr> [<comment>]
58///
59///     <blank><rr> [<comment>]
60///
61/// Blank lines, with or without comments, are allowed anywhere in the file.
62///
63/// Two control entries are defined: $ORIGIN and $INCLUDE.  $ORIGIN is
64/// followed by a domain name, and resets the current origin for relative
65/// domain names to the stated name.  $INCLUDE inserts the named file into
66/// the current file, and may optionally specify a domain name that sets the
67/// relative domain name origin for the included file.  $INCLUDE may also
68/// have a comment.  Note that a $INCLUDE entry never changes the relative
69/// origin of the parent file, regardless of changes to the relative origin
70/// made within the included file.
71///
72/// The last two forms represent RRs.  If an entry for an RR begins with a
73/// blank, then the RR is assumed to be owned by the last stated owner.  If
74/// an RR entry begins with a <domain-name>, then the owner name is reset.
75///
76/// <rr> contents take one of the following forms:
77///
78///     [<TTL>] [<class>] <type> <RDATA>
79///
80///     [<class>] [<TTL>] <type> <RDATA>
81///
82/// The RR begins with optional TTL and class fields, followed by a type and
83/// RDATA field appropriate to the type and class.  Class and type use the
84/// standard mnemonics, TTL is a decimal integer.  Omitted class and TTL
85/// values are default to the last explicitly stated values.  Since type and
86/// class mnemonics are disjoint, the parse is unique.  (Note that this
87/// order is different from the order used in examples and the order used in
88/// the actual RRs; the given order allows easier parsing and defaulting.)
89///
90/// <domain-name>s make up a large share of the data in the Zone File.
91/// The labels in the domain name are expressed as character strings and
92/// separated by dots.  Quoting conventions allow arbitrary characters to be
93/// stored in domain names.  Domain names that end in a dot are called
94/// absolute, and are taken as complete.  Domain names which do not end in a
95/// dot are called relative; the actual domain name is the concatenation of
96/// the relative part with an origin specified in a $ORIGIN, $INCLUDE, or as
97/// an argument to the Zone File loading routine.  A relative name is an
98/// error when no origin is available.
99///
100/// <character-string> is expressed in one or two ways: as a contiguous set
101/// of characters without interior spaces, or as a string beginning with a "
102/// and ending with a ".  Inside a " delimited string any character can
103/// occur, except for a " itself, which must be quoted using \ (back slash).
104///
105/// Because these files are text files several special encodings are
106/// necessary to allow arbitrary data to be loaded.  In particular:
107///
108///                 of the root.
109///
110/// @               A free standing @ is used to denote the current origin.
111///
112/// \X              where X is any character other than a digit (0-9), is
113///                 used to quote that character so that its special meaning
114///                 does not apply.  For example, "\." can be used to place
115///                 a dot character in a label.
116///
117/// \DDD            where each D is a digit is the octet corresponding to
118///                 the decimal number described by DDD.  The resulting
119///                 octet is assumed to be text and is not checked for
120///                 special meaning.
121///
122/// ( )             Parentheses are used to group data that crosses a line
123///                 boundary.  In effect, line terminations are not
124///                 recognized within parentheses.
125///
126/// ;               Semicolon is used to start a comment; the remainder of
127///                 the line is ignored.
128/// ```
129pub struct Parser<'a> {
130    lexers: Vec<(Lexer<'a>, Option<PathBuf>)>,
131    origin: Option<Name>,
132}
133
134impl<'a> Parser<'a> {
135    /// Returns a new Zone file parser
136    ///
137    /// The `path` argument's parent directory is used to resolve relative `$INCLUDE` paths.
138    /// Relative `$INCLUDE` paths will yield an error if `path` is `None`.
139    pub fn new(
140        input: impl Into<Cow<'a, str>>,
141        path: Option<PathBuf>,
142        mut origin: Option<Name>,
143    ) -> Self {
144        if let Some(origin) = &mut origin {
145            origin.set_fqdn(true);
146        }
147        Self {
148            lexers: vec![(Lexer::new(input), path)],
149            origin,
150        }
151    }
152
153    /// Parse a file from the Lexer
154    ///
155    /// # Return
156    ///
157    /// A pair of the Zone origin name and a map of all Keys to RecordSets
158    pub fn parse(mut self) -> ParseResult<(Name, BTreeMap<RrKey, RecordSet>)> {
159        let mut cx = Context::new(self.origin);
160        let mut state = State::StartLine;
161        let mut stack = self.lexers.len();
162
163        'outer: while let Some((lexer, path)) = self.lexers.last_mut() {
164            while let Some(t) = lexer.next_token()? {
165                state = match state {
166                    State::StartLine => {
167                        // current_name is not reset on the next line b/c it might be needed from the previous
168                        cx.rtype = None;
169
170                        match t {
171                            // if Dollar, then $INCLUDE or $ORIGIN
172                            Token::Include => State::Include(None),
173                            Token::Origin => State::Origin,
174                            Token::Ttl => State::Ttl,
175
176                            // if CharData, then Name then ttl_class_type
177                            Token::CharData(data) => {
178                                cx.current_name = Some(Name::parse(&data, cx.origin.as_ref())?);
179                                State::TtlClassType
180                            }
181
182                            // @ is a placeholder for specifying the current origin
183                            Token::At => {
184                                cx.current_name.clone_from(&cx.origin); // TODO a COW or RC would reduce copies...
185                                State::TtlClassType
186                            }
187
188                            // if blank, then nothing or ttl_class_type
189                            Token::Blank => State::TtlClassType,
190                            Token::EOL => State::StartLine, // probably a comment
191                            _ => return Err(ParseErrorKind::UnexpectedToken(t).into()),
192                        }
193                    }
194                    State::Ttl => match t {
195                        Token::CharData(data) => {
196                            cx.ttl = Some(Self::parse_time(&data)?);
197                            State::StartLine
198                        }
199                        _ => return Err(ParseErrorKind::UnexpectedToken(t).into()),
200                    },
201                    State::Origin => {
202                        match t {
203                            Token::CharData(data) => {
204                                // TODO an origin was specified, should this be legal? definitely confusing...
205                                cx.origin = Some(Name::parse(&data, None)?);
206                                State::StartLine
207                            }
208                            _ => return Err(ParseErrorKind::UnexpectedToken(t).into()),
209                        }
210                    }
211                    State::Include(include_path) => match (t, include_path) {
212                        (Token::CharData(data), None) => State::Include(Some(data)),
213                        (Token::EOL, Some(include_path)) => {
214                            // RFC1035 (section 5) does not specify how filename for $INCLUDE
215                            // should be resolved into file path. The underlying code implements the
216                            // following:
217                            // * if the path is absolute (relies on Path::is_absolute), it uses normalized path
218                            // * otherwise, it joins the path with parent root of the current file
219                            //
220                            // TODO: Inlining files specified using non-relative path might potentially introduce
221                            // security issue in some cases (e.g. when working with zone files from untrusted sources)
222                            // and should probably be configurable by user.
223
224                            if stack > MAX_INCLUDE_LEVEL {
225                                return Err(ParseErrorKind::Message(
226                                    "Max depth level for nested $INCLUDE is reached",
227                                )
228                                .into());
229                            }
230
231                            let include = Path::new(&include_path);
232                            let include = match (include.is_absolute(), path) {
233                                (true, _) => include.to_path_buf(),
234                                (false, Some(path)) => path
235                                    .parent()
236                                    .expect("file has to have parent folder")
237                                    .join(include),
238                                (false, None) => {
239                                    return Err(ParseErrorKind::Message(
240                                        "Relative $INCLUDE is not supported",
241                                    )
242                                    .into());
243                                }
244                            };
245
246                            let input = fs::read_to_string(&include)?;
247                            let lexer = Lexer::new(input);
248                            self.lexers.push((lexer, Some(include)));
249                            stack += 1;
250                            state = State::StartLine;
251                            continue 'outer;
252                        }
253                        (Token::CharData(_), Some(_)) => {
254                            return Err(ParseErrorKind::Message(
255                                "Domain name for $INCLUDE is not supported",
256                            )
257                            .into());
258                        }
259                        (t, _) => {
260                            return Err(ParseErrorKind::UnexpectedToken(t).into());
261                        }
262                    },
263                    State::TtlClassType => {
264                        match t {
265                            // if number, TTL
266                            // Token::Number(num) => ttl = Some(*num),
267                            // One of Class or Type (these cannot be overlapping!)
268                            Token::CharData(mut data) => {
269                                // if it's a number it's a ttl
270                                let result: ParseResult<u32> = Self::parse_time(&data);
271                                if let Ok(ttl) = result {
272                                    cx.ttl = Some(ttl);
273                                    State::TtlClassType // hm, should this go to just ClassType?
274                                } else {
275                                    // if can parse DNSClass, then class
276                                    data.make_ascii_uppercase();
277                                    let result = DNSClass::from_str(&data);
278                                    if let Ok(parsed) = result {
279                                        cx.class = parsed;
280                                        State::TtlClassType
281                                    } else {
282                                        // if can parse RecordType, then RecordType
283                                        cx.rtype = Some(RecordType::from_str(&data)?);
284                                        State::Record(vec![])
285                                    }
286                                }
287                            }
288                            // could be nothing if started with blank and is a comment, i.e. EOL
289                            Token::EOL => {
290                                State::StartLine // next line
291                            }
292                            _ => return Err(ParseErrorKind::UnexpectedToken(t).into()),
293                        }
294                    }
295                    State::Record(record_parts) => {
296                        // b/c of ownership rules, perhaps, just collect all the RData components as a list of
297                        //  tokens to pass into the processor
298                        match t {
299                            Token::EOL => {
300                                cx.insert(record_parts)?;
301                                State::StartLine
302                            }
303                            Token::CharData(part) => {
304                                let mut record_parts = record_parts;
305                                record_parts.push(part);
306                                State::Record(record_parts)
307                            }
308                            // TODO: we should not tokenize the list...
309                            Token::List(list) => {
310                                let mut record_parts = record_parts;
311                                record_parts.extend(list);
312                                State::Record(record_parts)
313                            }
314                            _ => return Err(ParseErrorKind::UnexpectedToken(t).into()),
315                        }
316                    }
317                };
318            }
319
320            // Extra flush at the end for the case of missing endline
321            if let State::Record(record_parts) = mem::replace(&mut state, State::StartLine) {
322                cx.insert(record_parts)?;
323            }
324
325            stack -= 1;
326            self.lexers.pop();
327        }
328
329        //
330        // build the Authority and return.
331        let origin = cx.origin.ok_or_else(|| {
332            ParseError::from(ParseErrorKind::Message("$ORIGIN was not specified"))
333        })?;
334        Ok((origin, cx.records))
335    }
336
337    /// parses the string following the rules from:
338    ///  <https://tools.ietf.org/html/rfc2308> (NXCaching RFC) and
339    ///  <https://www.zytrax.com/books/dns/apa/time.html>
340    ///
341    /// default is seconds
342    /// #s = seconds = # x 1 seconds (really!)
343    /// #m = minutes = # x 60 seconds
344    /// #h = hours   = # x 3600 seconds
345    /// #d = day     = # x 86400 seconds
346    /// #w = week    = # x 604800 seconds
347    ///
348    /// returns the result of the parsing or and error
349    ///
350    /// # Example
351    /// ```
352    /// use hickory_proto::serialize::txt::Parser;
353    ///
354    /// assert_eq!(Parser::parse_time("0").unwrap(),  0);
355    /// assert!(Parser::parse_time("s").is_err());
356    /// assert!(Parser::parse_time("").is_err());
357    /// assert_eq!(Parser::parse_time("0s").unwrap(), 0);
358    /// assert_eq!(Parser::parse_time("1").unwrap(),  1);
359    /// assert_eq!(Parser::parse_time("1S").unwrap(), 1);
360    /// assert_eq!(Parser::parse_time("1s").unwrap(), 1);
361    /// assert_eq!(Parser::parse_time("1M").unwrap(), 60);
362    /// assert_eq!(Parser::parse_time("1m").unwrap(), 60);
363    /// assert_eq!(Parser::parse_time("1H").unwrap(), 3600);
364    /// assert_eq!(Parser::parse_time("1h").unwrap(), 3600);
365    /// assert_eq!(Parser::parse_time("1D").unwrap(), 86400);
366    /// assert_eq!(Parser::parse_time("1d").unwrap(), 86400);
367    /// assert_eq!(Parser::parse_time("1W").unwrap(), 604800);
368    /// assert_eq!(Parser::parse_time("1w").unwrap(), 604800);
369    /// assert_eq!(Parser::parse_time("1s2d3w4h2m").unwrap(), 1+2*86400+3*604800+4*3600+2*60);
370    /// assert_eq!(Parser::parse_time("3w3w").unwrap(), 3*604800+3*604800);
371    /// assert!(Parser::parse_time("7102w").is_err());
372    /// ```
373    pub fn parse_time(ttl_str: &str) -> ParseResult<u32> {
374        if ttl_str.is_empty() {
375            return Err(ParseErrorKind::ParseTime(ttl_str.to_string()).into());
376        }
377
378        let (mut state, mut value) = (None, 0_u32);
379        for (i, c) in ttl_str.char_indices() {
380            let start = match (state, c) {
381                (None, '0'..='9') => {
382                    state = Some(i);
383                    continue;
384                }
385                (Some(_), '0'..='9') => continue,
386                (Some(start), 'S' | 's' | 'M' | 'm' | 'H' | 'h' | 'D' | 'd' | 'W' | 'w') => start,
387                _ => return Err(ParseErrorKind::ParseTime(ttl_str.to_string()).into()),
388            };
389
390            // All allowed chars are ASCII, so using char indexes to slice &[u8] is OK
391            let number = u32::from_str(&ttl_str[start..i])
392                .map_err(|_| ParseErrorKind::ParseTime(ttl_str.to_string()))?;
393
394            let multiplier = match c {
395                'S' | 's' => 1,
396                'M' | 'm' => 60,
397                'H' | 'h' => 3_600,
398                'D' | 'd' => 86_400,
399                'W' | 'w' => 604_800,
400                _ => unreachable!(),
401            };
402
403            value = number
404                .checked_mul(multiplier)
405                .and_then(|add| value.checked_add(add))
406                .ok_or_else(|| ParseErrorKind::ParseTime(ttl_str.to_string()))?;
407
408            state = None;
409        }
410
411        if let Some(start) = state {
412            // All allowed chars are ASCII, so using char indexes to slice &[u8] is OK
413            let number = u32::from_str(&ttl_str[start..])
414                .map_err(|_| ParseErrorKind::ParseTime(ttl_str.to_string()))?;
415            value = value
416                .checked_add(number)
417                .ok_or_else(|| ParseErrorKind::ParseTime(ttl_str.to_string()))?;
418        }
419
420        Ok(value)
421    }
422}

424struct Context {
425    origin: Option<Name>,
426    records: BTreeMap<RrKey, RecordSet>,
427    class: DNSClass,
428    current_name: Option<Name>,
429    rtype: Option<RecordType>,
430    ttl: Option<u32>,
431}
432
433impl Context {
434    fn new(origin: Option<Name>) -> Self {
435        Self {
436            origin,
437            records: BTreeMap::default(),
438            class: DNSClass::IN,
439            current_name: None,
440            rtype: None,
441            ttl: None,
442        }
443    }
444
445    fn insert(&mut self, record_parts: Vec<String>) -> ParseResult<()> {
446        // call out to parsers for difference record types
447        // all tokens as part of the Record should be chardata...
448        let rtype = self
449            .rtype
450            .ok_or_else(|| ParseError::from("record type not specified"))?;
451
452        let rdata = RData::parse(
453            rtype,
454            record_parts.iter().map(AsRef::as_ref),
455            self.origin.as_ref(),
456        )?;
457
458        // verify that we have everything we need for the record
459        // TODO COW or RC would reduce mem usage, perhaps Name should have an intern()...
460        //  might want to wait until RC.weak() stabilizes, as that would be needed for global
461        //  memory where you want
462        let mut name = self
463            .current_name
464            .clone()
465            .ok_or_else(|| ParseError::from("record name not specified"))?;
466
467        // slightly annoying, need to grab the TTL, then move rdata into the record,
468        //  then check the Type again and have custom add logic.
469        let set_ttl = match (rtype, self.ttl, &rdata) {
470            // TTL for the SOA is set internally...
471            // expire is for the SOA, minimum is default for records
472            (RecordType::SOA, _, RData::SOA(soa)) => {
473                // TODO, this looks wrong, get_expire() should be get_minimum(), right?
474                let set_ttl = soa.expire() as u32; // the spec seems a little inaccurate with u32 and i32
475                if self.ttl.is_none() {
476                    self.ttl = Some(soa.minimum());
477                } // TODO: should this only set it if it's not set?
478                set_ttl
479            }
480            (RecordType::SOA, _, _) => {
481                return ParseResult::Err(ParseError::from(format!(
482                    "invalid RData here, expected SOA: {rdata:?}"
483                )));
484            }
485            (_, Some(ttl), _) => ttl,
486            (_, None, _) => return Err(ParseError::from("record ttl not specified")),
487        };
488
489        // TODO: validate record, e.g. the name of SRV record allows _ but others do not.
490
491        // move the rdata into record...
492        name.set_fqdn(true);
493        let mut record = Record::from_rdata(name, set_ttl, rdata);
494        record.set_dns_class(self.class);
495
496        // add to the map
497        let entry = self.records.entry(RrKey::new(
498            LowerName::new(record.name()),
499            record.record_type(),
500        ));
501        match (rtype, entry) {
502            (RecordType::SOA, Entry::Occupied(_)) => {
503                return Err(ParseError::from("SOA is already specified"));
504            }
505            (_, Entry::Vacant(entry)) => {
506                entry.insert(RecordSet::from(record));
507            }
508            (_, Entry::Occupied(mut entry)) => {
509                entry.get_mut().insert(record, 0);
510            }
511        };
512
513        Ok(())
514    }
515}
516
517enum State {
518    StartLine,    // start of line, @, $<WORD>, Name, Blank
519    TtlClassType, // [<TTL>] [<class>] <type>,
520    Ttl,          // $TTL <time>
521    Record(Vec<String>),
522    Include(Option<String>), // $INCLUDE <filename>
523    Origin,
524}
